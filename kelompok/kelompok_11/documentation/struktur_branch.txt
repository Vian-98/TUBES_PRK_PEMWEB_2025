# Struktur Branch dan Workflow Git

## Branch Strategy
Proyek ini menggunakan **Feature Branch Workflow** dimana setiap anggota bekerja di branch terpisah berdasarkan fitur.

### Branch untuk Anggota 4:
1. `feature/inventory` - Modul inventory (sparepart & supplier CRUD)
2. `feature/dashboard` - Modul dashboard dengan AJAX
3. `feature/ui-docs` - UI layout dan dokumentasi

## Workflow Git

### 1. Mulai Fitur Baru
```bash
# Checkout dari master
git checkout master
git pull origin master

# Buat branch baru
git checkout -b feature/nama-fitur
```

### 2. Development
```bash
# Cek status
git status

# Add files
git add .

# Commit dengan pesan jelas
git commit -m "feat: tambah fitur X"
```

### 3. Commit Message Convention
Gunakan prefix berikut:
- `feat:` - Fitur baru
- `fix:` - Bug fix
- `docs:` - Perubahan dokumentasi
- `style:` - Formatting, whitespace
- `refactor:` - Refactoring code
- `test:` - Tambah/update tests
- `chore:` - Maintenance

**Contoh:**
```bash
git commit -m "feat: tambah CRUD sparepart"
git commit -m "fix: perbaiki query low stock"
git commit -m "docs: update README dengan panduan instalasi"
```

### 4. Push ke Remote
```bash
# Push branch pertama kali
git push -u origin feature/nama-fitur

# Push selanjutnya
git push
```

### 5. Merge ke Master
```bash
# Pastikan branch up to date
git checkout feature/nama-fitur
git rebase master

# Checkout ke master
git checkout master
git merge feature/nama-fitur

# Push ke remote
git push origin master
```

## Best Practices

### DO:
✅ Commit dengan pesan yang jelas dan deskriptif
✅ Commit secara bertahap (jangan batch besar)
✅ Pull sebelum push untuk avoid conflicts
✅ Test sebelum commit
✅ Review code sebelum merge

### DON'T:
❌ Commit langsung ke master
❌ Push code yang error
❌ Gunakan pesan commit generic ("update", "fix")
❌ Commit file yang tidak perlu (.env, node_modules)
❌ Force push tanpa koordinasi

## Git Commands Reference

### Basic Commands
```bash
git status              # Cek status working directory
git log --oneline       # Lihat commit history
git diff                # Lihat perubahan
git branch              # List branches
git branch -d feature/x # Hapus branch
```

### Undo Changes
```bash
git checkout -- file    # Undo perubahan file
git reset HEAD file     # Unstage file
git reset --soft HEAD~1 # Undo commit (keep changes)
git reset --hard HEAD~1 # Undo commit (discard changes)
```

### Remote Operations
```bash
git remote -v           # Lihat remote URLs
git fetch origin        # Fetch dari remote
git pull origin master  # Pull dari remote
git push origin branch  # Push ke remote
```

## Conflict Resolution
Jika terjadi conflict saat merge:

```bash
# 1. Identifikasi file conflict
git status

# 2. Edit file, hapus conflict markers
# <<<<<<< HEAD
# >>>>>>> feature-branch

# 3. Add resolved files
git add .

# 4. Commit merge
git commit -m "merge: resolve conflicts"
```

## Rollback Strategy
Jika ada masalah setelah merge:

```bash
# Lihat commit history
git reflog

# Rollback ke commit tertentu
git reset --hard commit-hash

# Atau buat branch baru dari commit lama
git checkout -b recovery commit-hash
```

## Branch Protection (Production)
Untuk production, setup branch protection:
- Require pull request reviews
- Require status checks to pass
- Restrict who can push
- Require linear history
